# 終了とシャットダウンのフロー (tui)

このドキュメントは、Rust TUI (`codex-rs/tui`) における終了、シャットダウン、および割り込みの動作について説明する。
今後の終了 / シャットダウン変更について検討する際の、Codex 開発者および Codex 自身向けの資料である。

このドキュメントは、これまで別々に存在していた履歴と設計ノートを置き換える。ハイレベルな履歴は以下に要約してあり、詳細は PR #8936 に記録されている。

## 用語

- **Exit**: UI のイベントループを終了し、プロセスを終了する。
- **Shutdown**: グレースフルなエージェント / コアのシャットダウン (`Op::Shutdown`) を要求し、
  クリーンアップ処理が実行できるよう `ShutdownComplete` を待つ。
- **Interrupt**: 実行中のオペレーションをキャンセルする (`Op::Interrupt`)。

## イベントモデル (AppEvent)

Exit は、明示的なモードを持つ単一のイベントを通じて調整される。

- `AppEvent::Exit(ExitMode::ShutdownFirst)`
  - クリーンアップが実行されるよう、ユーザー操作による終了ではこれを優先して使用する。
- `AppEvent::Exit(ExitMode::Immediate)`
  - 即時終了のためのエスケープハッチ。これはシャットダウンをバイパスし、進行中の作業（例: タスク、ロールアウトのフラッシュ、子プロセスのクリーンアップ）が失われる可能性がある。

`App` がコーディネーターであり、`Op::Shutdown` を送信し、`ExitMode::Immediate` が到着したときにのみ
UI ループを抜ける（通常は `ShutdownComplete` の後）。

## ユーザー起因の終了フロー

### Ctrl+C

UI レイヤーでの優先順位は次のとおり:

1. アクティブなモーダル / ビューが最初に処理する機会を得る (`BottomPane::on_ctrl_c`)。
   - モーダルがそれを処理した場合、終了フローはそこで停止する。
   - モーダル / ポップアップが Ctrl+C を処理した場合、終了ショートカットはクリアされる。そのため、
     モーダルを閉じた後の Ctrl+C が誤って終了用にアームされることはない。
2. ユーザーがすでに Ctrl+C をアームしていて 1 秒の猶予時間がまだ切れていない場合、
   2 回目の Ctrl+C で即座にシャットダウン優先の終了が行われる。
3. それ以外の場合、`ChatWidget` は Ctrl+C をアームし、1 秒間終了ヒント
   (`ctrl + c again to quit`) を表示する。
4. キャンセル可能な処理（ストリーミング / tools / review）がアクティブな場合、
   `ChatWidget` は `Op::Interrupt` を送信する。

### Ctrl+D

- 入力コンポーザーが空で **かつ** モーダルがアクティブでない場合にのみ、終了フローに参加する。
  - 最初の押下時には、終了ヒント（Ctrl+C と同じ）を表示し、1 秒タイマーを開始する。
  - ヒントが表示されている間にもう一度押された場合、シャットダウン優先の終了を要求する。
- いずれかのモーダル / ポップアップが開いている場合、キーイベントはビューにルーティングされ、
  Ctrl+D は終了を試みない。

### スラッシュコマンド

- `/quit`, `/exit`, `/logout` はプロンプト **なし** でシャットダウン優先の終了を要求する。
  スラッシュコマンドは誤って実行されにくく、終了したいという明確な意図を示すためである。

### /new

- 終了せずにシャットダウンのみを使用する（`ShutdownComplete` を抑制する）ことで、アプリが
  終了することなく新しいセッションを開始できるようにする。

## シャットダウン完了と抑制

`ShutdownComplete` は、コアのクリーンアップが完了したことを示すシグナルである。UI はこれを終了の境界として扱う。

- `ChatWidget` は `ShutdownComplete` で `Exit(Immediate)` を要求する。
- シャットダウンがクリーンアップ手順として使用される場合（例: `/new`）、`App` は 1 回分の
  `ShutdownComplete` を抑制できる。

## エッジケースと不変条件

- **Review mode** はキャンセル可能な処理として扱われる。Ctrl+C は終了ではなくレビューを中断すべきである。
- **モーダルが開いている** 状態では、モーダルが明示的に Ctrl+C を処理しないと判断しない限り、
  Ctrl+C / Ctrl+D で終了すべきではない。
- **即時終了** は通常のユーザーパスではなく、シャットダウン完了時のフォールバックや緊急終了のためのものだ。
  クリーンアップをスキップするため、使用は控えめにすること。

## テストに関する期待事項

最低限、次のケースをカバーしたい。

- 作業中に Ctrl+C を押すと終了ではなく割り込みが発生する。
- アイドル状態かつ空のときに Ctrl+C を押すと終了ヒントが表示され、2 回目の押下で
  シャットダウン優先の終了が行われる。
- モーダルが開いている状態での Ctrl+D は終了しない。
- `/quit` / `/exit` / `/logout` はプロンプトなしで終了するが、依然としてシャットダウン優先である。
  - アイドル状態かつ空のときに Ctrl+D を押すと終了ヒントが表示され、2 回目の押下で
    シャットダウン優先の終了が行われる。

## 履歴（概要）

Codex では歴史的に、段階的な変更や状態管理のリグレッションにより、終了ジェスチャーごとに
「即時終了」と「シャットダウン優先」が混在していた。このドキュメントは、現在のシャットダウン優先の
統一されたアプローチを反映している。詳細な履歴とその理由については PR #8936 を参照のこと。
