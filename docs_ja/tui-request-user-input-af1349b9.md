# ユーザー入力要求オーバーレイ (TUI)

このメモでは、TUI オーバーレイについて説明します。そのオーバーレイは、
`RequestUserInputEvent` の回答を収集するために使用されます。

## 概要

このオーバーレイは一度に 1 つの質問を表示し、次の情報を収集します:

- オプションが存在する場合の、選択されたオプション 1 件。
- 自由形式のノート（常に利用可能）。

オプションが存在する場合、ノートは選択されたオプションごとに保存され、
さらに最初のオプションがデフォルトで選択されるため、すべてのオプション質問には必ず回答が存在します。質問にオプションがなく、ノートも入力されなかった場合、その回答は
`skipped` として送信されます。

## フォーカスと入力のルーティング

このオーバーレイは、簡単なフォーカス状態を管理します:

- **オプション**: Up/Down で選択を移動し、Space で選択します。
- **ノート**: テキスト入力により、現在選択されているオプションのノートを編集します。

オプションにフォーカスがある状態でキー入力を行うと、自動的にノート入力に切り替わり、
自由形式の入力に対する摩擦を減らします。

## ナビゲーション

- Enter で次の質問に進みます。
- 最後の質問で Enter を押すと、すべての回答を送信します。
- PageUp/PageDown で質問間を移動します（複数の質問がある場合）。
- Esc はオプション選択モード中の実行を中断します。
- オプションの質問でノートが開いている場合、Tab または Esc でノートをクリアし、
  オプション選択に戻ります。

## レイアウトの優先順位

レイアウトは、質問とすべてのオプションが表示された状態を優先します。ノートと
フッターのヒントは利用可能なスペースが小さくなるにつれて折りたたまれ、ノートはスペースが限られた端末では
1 行の "Notes: ..." 入力にフォールバックします。
